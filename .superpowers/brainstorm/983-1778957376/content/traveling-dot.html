Traveling Dot — Which Behavior?

The dot moves along the SVG edges connecting income to goals. Pick the behavior that feels right.

A

Always On — Continuous Loop

Identical blue dots travel all edges at a steady pace, looping forever. Canvas always feels alive, no interaction needed.

₹ Income
🏠 Home
✈ Travel
🎓 Edu

Pros

  • No interaction needed
  • Instantly shows data flow metaphor

Cons

  • Could feel busy with many goals
B

Priority-Weighted — Speed + Size

Dot speed and size encode priority: P1 goal gets a big fast dot, P3 gets a small slow one. Visual hierarchy matches the data.

₹ Income
🏠 P1
✈ P2
🎓 P3

Pros

  • Encodes priority visually
  • Richer information density

Cons

  • More complex to implement
  • Can look chaotic with 5+ goals
C

Triggered — On Node Click Only

Dots fire outward from the income node only when you click a goal — one burst per interaction. Canvas stays calm by default.

← click a goal node → dot fires once along that edge

Pros

  • Clean, calm by default
  • Interaction has satisfying feedback

Cons

  • Less "alive" than always-on
