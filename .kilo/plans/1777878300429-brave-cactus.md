## Finding 3: Dead Google Fonts Import in `fonts.css`

**Location:** `fonts.css:1`

```css
@import url('https://fonts.googleapis.com/css2?family=Montserrat:wght@400;500;600;700;800&family=Space+Grotesk:wght@400;500;600;700&display=swap');
```

**Issue:** Downloads 7 weight variants of Montserrat + Space Grotesk (~150KB+) on every page load, but these are only FALLBACK fonts in the font stack. Lufga is loaded locally (5 .ttf files, line 3-35) and will resolve first in all cases since the `@font-face` declarations load from local files without a network dependency.

The only scenario where Montserrat/Space Grotesk would render is if ALL 5 Lufga .ttf files fail to load. This is effectively dead weight on every page load.

**Action:** Replace the `@import` with a local-only font stack. Keep the font-family variables pointing to `"Lufga", system-ui, sans-serif` (remove Montserrat and Space Grotesk from the fallback chain since they're unused without the import).

## Finding 4: Hardcoded Hex `#64748b` in `SankeyFlow.tsx`

**Location:** `src/app/components/SankeyFlow.tsx:28`

```ts
slate: '#64748b',
```

**Issue:** `#64748b` is not a design token variable. It's used as fallback when `--slate` CSS var doesn't exist. The dark mode `--tertiary` is `#64748b`. This should reference the CSS variable instead.

**Action:** Change to `resolveCssVar('--tertiary')` for proper light/dark mode support, with `'#64748b'` as the hardcoded fallback inside `resolveCssVar`.

## Finding 10: `--spacing` Token Inconsistency

**Location:** `theme.css:209,346`

```css
--spacing: 0.25rem;
```

**Issue:** The spacing scale is defined as 8px increments (lines 10-17: `--space-1` through `--space-12` = 8px * N). But `--spacing: 0.25rem` (= 4px) breaks the 8px grid. This is a Tailwind v4 compatibility requirement (Tailwind v4 reads `--spacing` for its spacing scale). However, the custom gap utilities (lines 631-642) reference `var(--space-*)` which is the 8px scale, while Tailwind itself uses the 4px-based `--spacing`. This creates two parallel spacing systems.

**Action:** Document this as intentional (Tailwind v4 compatibility). The `--space-*` custom tokens are for the custom gap utilities (which are unused anyway — see Finding 8). Once gap utilities are removed, the `--space-*` tokens are only used in `.bento-card` and `.pill-button` paddings. These could potentially be migrated to Tailwind classes too, but that's out of scope for this audit.
